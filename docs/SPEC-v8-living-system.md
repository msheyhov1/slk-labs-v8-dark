# CLAUDE.md — SLK-labs v8 «Living System»

## Роль

Ты — ведущий креативный инженер студии SLK-labs. Твоя задача — превратить существующий сайт (https://msheyhov1.github.io/slk-labs-v7/) в v8: сайт-доказательство. Не редизайн ради красоты, а инженерная демонстрация.

## Единственная мысль, которую должен вынести посетитель

> «Эти люди умеют делать невероятные вещи. Они шарят в технологиях на уровне, который я нигде больше не видел.»

Сайт не должен *рассказывать* про живые системы — он должен *быть* живой системой. Каждая wow-фича обязана быть настоящей: реальные данные, реальный realtime, реальные метрики. Никаких фейковых счётчиков и муляжей — техническая аудитория фейк чует мгновенно, и это убивает всё доверие.

## Что сохраняем из v7

- Вербальная идентичность: «Процессы, которые живут без вас», манифест, тон.
- Знак ensō, редакционная концепция «Ink · Dawn».
- Структура: Hero → Работы (3 кейса) → Услуги → Манифест → Контакт.
- Кейсы: New Link Food (web · 3d · i18n), Смена (Telegram Mini App), Дозор (TRC20 AML-мониторинг).
- Бюджеты качества: 60 FPS, Core Web Vitals в зелёной зоне, WCAG AA.

## Стек и хостинг

- **Next.js 15 (App Router) + TypeScript**, деплой на **Vercel** (не GitHub Pages — нужны edge-функции и realtime).
- WebGL/шейдеры: чистый WebGL2 или ogl (лёгкий); three.js только если реально нужен.
- Realtime-присутствие: **PartyKit** (предпочтительно) или Supabase Realtime.
- Edge/API: Vercel Edge Functions / Route Handlers.
- Анимации: CSS scroll-driven animations + View Transitions API там, где поддерживается; GSAP только для оркестрации сложных сцен.
- Никаких тяжёлых UI-китов. Бюджет JS первой загрузки: ≤ 150 KB gzip (WebGL-сцена и терминал — lazy, по мере скролла/вызова).

## Фичи

### Tier 1 — обязательные (ядро концепции)

**1. Живая телеметрия — HUD «жизненные показатели»**
Плавающий сворачиваемый HUD (уголок экрана) с настоящими метриками:
- FPS в реальном времени (requestAnimationFrame-счётчик);
- вес страницы и время загрузки (Performance API: transferSize, LCP, TTFB);
- устройство посетителя: GPU (WEBGL_debug_renderer_info), ядра CPU (hardwareConcurrency), тип сети (Network Information API, с graceful fallback);
- количество посетителей на сайте прямо сейчас (см. фичу 3).
Подача: моноширинный шрифт, стиль бортового самописца. Подпись-панч: «Ваш {GPU} · держим 60 fps». Это одновременно и флекс, и доказательство перфоманса.

**2. Командная палитра (Cmd+K / кнопка) + терминал-режим**
- Палитра: навигация, действия («Написать в Telegram», «Скопировать почту», «Переключить тему»), поиск по кейсам.
- Внутри палитры — режим терминала: посетитель печатает команды, сайт отвечает. Минимальный набор: `help`, `ls works`, `open <кейс>`, `status` (реальный uptime, см. фичу 4), `whoami` (телеметрия посетителя), `hire` (открывает контакт), `enso --reseed` (перерисовать знак). 1–2 пасхалки на усмотрение.
- Ответы стилизованы под настоящий CLI: prompt-строка, задержки печати, коды выхода.

**3. Присутствие в реальном времени (настоящий realtime-бекенд)**
- Счётчик «Сейчас на сайте: N» — живой, через PartyKit-комнату.
- Полупрозрачные курсоры других посетителей в зоне hero/манифеста (throttle 30 Гц, интерполяция, максимум ~15 видимых курсоров, приватность: без никнеймов, только анонимные метки-точки).
- На мобильных вместо курсоров — «пульс»: точки-присутствия на мини-карте секций.

**4. Публичный статус реальных систем**
Секция или строка в футере: живые health-check'и реальных проектов студии (боты, API — список эндпоинтов задать через env). Edge-функция пингует их раз в 60 сек, кэширует, отдаёт: uptime, latency, зелёный/красный статус. «Мы говорим, что наши процессы живут без нас — вот они, живые, проверьте.»

**5. Сеть связей, собирающая ensō (сигнатурная сцена)**
Фон hero — живая сеть: узлы и линии-связи (визуальный язык «процессов»: боты, API, платежи — подписи-теги на узлах). Узлы дрейфуют, связи пульсируют при передаче «пакетов». На загрузке (или по скроллу в hero) сеть стягивается и её линии собирают из себя знак ensō — штрих дорисовывается симуляцией кисти (давление, рваный край, брызги туши), затем сеть распадается обратно в фон и продолжает жить.
Seed — из характеристик устройства посетителя (без fingerprint-трекинга: только время + viewport + GPU-строка, хешированные локально). Подпись: «Этот оттиск собран для вас. Второго такого нет.» Команда `enso --reseed` пересобирает.
Тот же язык линий-связей — в переходах между секциями: линии «тянут» заголовки на место (см. «Полёт» ниже).

**6. Живые окна кейсов**
Клик по кейсу открывает не модалку с текстом, а стилизованное «окно ОС» (заголовок, кнопки, тень, лёгкий scale-in) с настоящим живым сайтом проекта внутри (iframe с `loading="lazy"`, sandbox, реальный URL, например New Link Food). Посетитель может скроллить и кликать сайт прямо в окне — работа не описана, работа открыта.
- Рядом с окном — колонка фактов: стек, сроки, метрики.
- Фоллбек: если сайт запрещает встраивание (X-Frame-Options/CSP) или это Telegram Mini App — короткая видео-запись экрана (muted, loop, ≤ 2 МБ, poster) в том же окне + кнопка «Открыть вживую».
- Окно можно «развернуть» на весь экран и перетаскивать на десктопе (небольшой драг — приятная деталь «настоящей ОС»).

**7. Контактные окна-сцены**
Каждый способ связи — интерактивная карточка с собственной микро-сценой на клик/тап:
- **Telegram**: из карточки складывается бумажный самолётик и улетает по кривой Безье за край экрана → открывается t.me-ссылка.
- **Email**: карточка складывается в конверт, штемпель «SLK» припечатывается → адрес скопирован, тост «В буфере».
- **Бриф через консьержа**: карточка «распечатывается» — раскрывается чат AI-консьержа (фича 9).
Каждая сцена ≤ 800 мс, играет один раз за сессию (повторный клик — сразу действие), при prefers-reduced-motion — мгновенное действие без сцены. Анимации — canvas/CSS, без lottie.

### Tier 2 — сильно желательные

**8. Живой кейс «Дозор»**
В карточке кейса — настоящий мини-виджет: живой курс USDT/RUB (публичный API, например Binance/OKX P2P или tickers, через edge-прокси с кэшем 30–60 сек) и спарклайн за час. Кейс не описывается — кейс работает.

**9. AI-консьерж**
Чат-виджет на Claude API через серверный route handler (ключ только на сервере). Системный промпт: консьерж студии — отвечает про услуги, кейсы, процесс, мягко ведёт к брифу; собранный бриф отправляет в Telegram студии (бот-нотификация). Rate-limit по IP на edge.

**10. Boot-sequence**
Первый заход: ~1.2–1.5 сек «пробуждение системы» — строки лога (реальные: версия сборки, время рендера, GPU посетителя), затем сборка ensō из сети (фича 5) и раскрытие hero. Повторные заходы — мгновенно (sessionStorage). Обязательно skip по клику и полное отключение при prefers-reduced-motion.

**11. «Полёт» — скролл как движение камеры**
Весь сайт ощущается как один непрерывный пролёт, а не смена статичных экранов:
- Скролл с инерцией (Lenis или собственная реализация) и оркестрованные scroll-driven сцены: заголовки секций «прилетают» — линии-связи из фоновой сети стягивают буквы на место; кейсы наезжают с лёгким параллаксом глубины.
- Кинетическая типографика в манифесте: ключевые слова на скорости, резкие остановки — ощущение скорости и точности одновременно.
- Через палитру/якоря — быстрый «перелёт» к секции: ускорение → торможение, а не мгновенный прыжок.
- Жёсткое правило: один цельный ритм движения на весь сайт, никакого набора разрозненных эффектов; всё на transform/opacity (композитинг), ничего, что роняет FPS ниже 60. При prefers-reduced-motion — обычный скролл.

### Tier 3 — если останется бюджет производительности

- Шейдерный фон hero: «тушь в воде», реагирует на курсор; на мобильных — на гироскоп (DeviceOrientation, по явному разрешению).
- Пасхалка в DevTools: ASCII-ensō и приглашение на работу в console.log.
- View Transitions между секциями/кейсами.

## Дизайн-рамка

- Основа — принятый «Ink · Dawn»: редакционная типографика постерного масштаба, тёплая бумага, глубокий печатный красный, много воздуха.
- Вся «техника» (HUD, терминал, статусы) — на контрасте: моноширинный шрифт, тонкие линии, как приборная панель поверх бумаги. Два мира — «печать» и «телеметрия» — и их стык и есть фирменный приём.
- Смелость тратим в одном месте: живой организм — сеть связей, собирающая ensō, единый «полёт» камеры и настоящая телеметрия. Всё остальное — тихо и дисциплинированно. Никакого зоопарка эффектов: каждая анимация принадлежит одному общему языку движения (линии-связи + инерция полёта).

## Бюджеты качества (жёсткие)

- Lighthouse: Performance ≥ 95 (mobile), A11y ≥ 95.
- 60 FPS на средних устройствах; на слабом GPU — авто-даунгрейд сцены (детект по renderer-строке и первому замеру FPS).
- prefers-reduced-motion: отключает boot-sequence, шейдеры, анимацию курсоров; ensō рисуется мгновенно.
- Полная работоспособность без JS-фич: контент, навигация и контакты доступны, HUD/терминал/realtime — прогрессивное улучшение.
- Клавиатурная навигация всего, включая палитру и терминал; видимый фокус.
- Всё на русском; архитектура i18n-ready (en — позже).

## Порядок работы

1. Каркас Next.js + перенос контента v7 + токены Ink · Dawn.
2. Сеть связей + сборка ensō + boot-sequence (сигнатурная сцена — сначала она).
3. «Полёт»: инерционный скролл + оркестрация сцен секций.
4. Живые окна кейсов (iframe + фоллбеки).
5. Телеметрия HUD.
6. Cmd+K палитра + терминал.
7. Контактные окна-сцены.
8. PartyKit-присутствие.
9. Status page + живой виджет «Дозора» (edge-функции).
10. AI-консьерж.
11. Полировка: перфоманс-аудит, reduced-motion, мобильные, скриншоты каждой секции и самокритика перед сдачей.

После каждого этапа — краткий отчёт: что сделано, реальные метрики, что срезано и почему.

## Чего не делать

- Фейковые данные и счётчики — ничего «нарисованного».
- Тяжёлые lottie/видео-фоны, сторонние виджеты чатов, куки-баннеры без нужды.
- Дефолтный AI-look (кремовый фон + терракотовый акцент, либо чёрный + кислотный акцент) — держаться утверждённых токенов Ink · Dawn.
- Ломать существующие URL-якоря (#works, #services, #contact).
